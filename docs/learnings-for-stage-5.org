* Design document is stale — delta from reality

The design document (=restly_ewf_design_document.org=) was written
before implementation and several sections no longer match. These are
the discrepancies discovered during stage 5.

** Decision 4 (Mounting) — mount features removed

Design says: =RESTLY_EWF_SERVER= owns =WSF_ROUTER= and offers
=mount_collection= / =mount_element= / =mount_resource=.

Reality: =RESTLY_EWF_SERVER= is a deferred execution mixin inheriting
=WSF_ROUTED_EXECUTION= + =WSF_ROUTED_URI_TEMPLATE_HELPER=. It has no
mount features — the app calls =map_uri_template_response= directly:

#+begin_src eiffel
setup_router
    local
        l_collection: RESTLY_EWF_COLLECTION_HANDLER
        l_element: RESTLY_EWF_ELEMENT_HANDLER
    do
        create l_collection.make (front)
        map_uri_template_response ("/todos", l_collection, Void)
        create l_element.make (front)
        map_uri_template_response ("/todos/{id}", l_element, Void)
    end
#+end_src

The mount signatures section and the EWF bridge cluster table both
refer to the old mount features and need updating.

* Key learnings

** WSF_ROUTED_SERVICE does not exist in EWF 25.12

The official video tutorial references =WSF_ROUTED_SERVICE= but it is
absent from Eiffel 25.12. The actual hierarchy:

#+begin_example
WSF_ROUTED (deferred: router)
  ├── WSF_ROUTED_EXECUTION (effective router, deferred setup_router)
  │     └── WSF_ROUTED_SKELETON_EXECUTION (adds proxy/unavailable checks)
  └── WSF_ROUTED_URI_TEMPLATE_HELPER (map_uri_template, map_uri_template_response)
#+end_example

=WSF_ROUTED_EXECUTION= is the modern equivalent — simpler than the
skeleton, no proxy/policy boilerplate needed.

** EWF standalone creates a new execution per request

State does not persist between requests unless stored in a =once=
feature. The old RESTLY_251021 used =once ("PROCESS")= for this.
Our =front= feature is =once= to share the pipeline across requests.

** Connection: close header required for single-threaded server

EWF standalone with =concurrency="none"= is single-threaded. Without
=Connection: close=, jsdom (used by the npm test suite) holds
connections open and the server blocks. All responses now include
this header via =add_cors_headers=.

** Preconditions may not fire across library boundaries

EiffelStudio assertion checking may not propagate into library code
at runtime. The =error_404_not_found= precondition on
=RESTLY_PROTOCOL.item= was not checked. Fix: explicit =has_key=
guards in the element handler before calling =storage [key]=.

** URI template extraction handles {id} automatically

=WSF_URI_TEMPLATE_MAPPING_I.try= matches the path, extracts
variables, and populates =req.path_parameters=. The handler reads
=req.path_parameter ("id")= — no manual string parsing needed.
The value is always a string; type conversion happens in
=RESTLY_INT_KEY_CONVERTER=.

** Router-level verb filtering would increase code, not reduce it

Two routes with verb dispatch inside the handler (current approach)
beats six =map_uri_template_response= calls with per-verb filtering.
OPTIONS handling is cleaner inside the handler too.
